# RVC SW Controller Functional and Non-Functional Requirements

## 1. Purpose

이 문서는 Robot Vacuum Cleaner(RVC) SW Controller의 요구사항을 기능 요구사항과 비기능 요구사항으로 구분해 정리한다.

현재 범위는 자동 청소 기능이다. 센서, 모터, 청소 장치 등 하드웨어 제어의 상세 설계와 구현은 제외하고, 소프트웨어 컨트롤러의 판단과 동작 흐름에 집중한다.

## 2. Functional Requirements

| ID | Requirement | Description |
| --- | --- | --- |
| FR-001 | 자동 청소 수행 | RVC는 가정용 바닥 표면을 자동으로 청소해야 한다. |
| FR-003 | 기본 직진 주행 | RVC는 장애물이 감지되지 않는 동안 청소하면서 직진해야 한다. |
| FR-004 | 전방 장애물 감지 대응 | RVC는 전방 장애물을 감지하면 현재 청소 진행을 멈춰야 한다. |
| FR-005 | 장애물 회피 방향 전환 | 전방 장애물 감지 후 RVC는 좌측 또는 우측으로 방향을 전환해야 한다. |
| FR-006 | 회피 후 청소 재개 | 방향 전환 후 RVC는 다시 전진하면서 청소를 재개해야 한다. |
| FR-007 | 삼면 장애물 대응 | 전방, 좌측, 우측에 모두 장애물이 있는 경우 RVC는 후진해야 한다. |
| FR-008 | 후진 후 방향 전환 | 삼면 장애물 상황에서 후진한 뒤 RVC는 좌측 또는 우측으로 방향을 전환해야 한다. |
| FR-009 | 후진 회피 후 청소 재개 | 후진 및 방향 전환 후 RVC는 다시 전진하면서 청소를 재개해야 한다. |
| FR-010 | 먼지 감지 | RVC는 먼지 감지 입력을 처리할 수 있어야 한다. |
| FR-011 | 청소 출력 증가 | RVC는 먼지를 감지하면 일정 시간 동안 청소 출력을 높여야 한다. |
| FR-012 | 청소 출력 복귀 | 먼지 대응 시간이 끝나면 RVC는 일반 청소 출력으로 복귀해야 한다. |

## 3. Non-Functional Requirements

| ID | Requirement | Description |
| --- | --- | --- |
| NFR-001 | 하드웨어 독립성 | 자동 청소 로직은 센서, 모터, 청소 장치의 상세 하드웨어 구현과 분리되어야 한다. |
| NFR-002 | 센서 확장성 | 향후 센서가 추가되거나 변경될 수 있으므로 센서 입력 처리는 확장 가능한 구조여야 한다. |
| NFR-003 | 동작 정책 변경 용이성 | 장애물 회피, 먼지 감지 대응, 이동 제어 판단 정책은 변경하기 쉬워야 한다. |
| NFR-004 | 유지보수성 | 자동 청소 로직은 이해하기 쉽고 수정 범위가 제한되도록 구성되어야 한다. |
| NFR-005 | 기능 확장성 | 특정 지점 선회 청소, 모바일 앱 통신, 머신러닝 기반 청소 판단 기능을 추후 추가할 수 있어야 한다. |
| NFR-006 | 설정 가능성 | 청소 출력 증가 시간, 출력 수준, 회피 방향 선택 정책 등은 추후 조정 가능해야 한다. |
| NFR-007 | 관심사 분리 | 센서 입력 해석, 청소 상태 판단, 이동 명령 생성은 서로 분리된 책임으로 다뤄져야 한다. |

## 4. Future Extension Candidates

다음 항목은 현재 필수 구현 범위는 아니지만 설계 시 확장 가능성을 고려한다.

- 센서 추가 또는 센서 종류 변경
- 특정 지점 선회 청소
- 모바일 앱과의 통신
- 머신러닝 및 추론 기반 청소 효율 개선

## 5. Out of Scope

- 센서 하드웨어의 상세 설계 및 구현
- 모터, 브러시 등 하드웨어 제어의 상세 구현
- 모바일 앱 자체 구현
- 머신러닝 모델의 학습 및 상세 구현
