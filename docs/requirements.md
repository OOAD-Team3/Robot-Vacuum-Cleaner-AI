# Robot Vacuum Cleaner SW Controller Requirements

## 1. Overview

Robot Vacuum Cleaner(RVC)는 가정용 바닥 표면을 자동으로 청소하고 물걸레질하는 소프트웨어 컨트롤러를 가진다.

이 문서는 현재까지 확인된 RVC SW Controller의 기능 요구사항, 비기능 요구사항, 범위 제외 항목, 향후 확장 고려사항을 정리한다.

## 2. Scope

현재 단계에서는 RVC의 자동 청소 기능에 집중한다.

상세한 하드웨어 제어 설계와 구현은 고려하지 않는다. 센서, 모터, 청소 장치 등의 실제 하드웨어 제어는 추상화된 입력과 출력으로 다룬다.

## 3. Functional Requirements

### 3.1 Automatic Cleaning

- RVC는 가정용 바닥 표면을 자동으로 청소할 수 있어야 한다.
- RVC는 청소 동작 중 물걸레질을 수행할 수 있어야 한다.
- RVC는 기본적으로 청소하면서 직진해야 한다.

### 3.2 Obstacle Handling

- RVC가 전방 장애물을 감지하면 청소를 멈춰야 한다.
- 전방 장애물 감지 후 RVC는 좌측 또는 우측으로 방향을 전환해야 한다.
- 방향 전환 후 RVC는 다시 전진하며 청소를 재개해야 한다.
- 전방, 좌측, 우측에 모두 장애물이 있는 경우 RVC는 후진해야 한다.
- 후진 후 RVC는 좌측 또는 우측으로 방향을 전환해야 한다.
- 방향 전환 후 RVC는 다시 전진하며 청소를 재개해야 한다.

### 3.3 Dust Detection

- RVC가 먼지를 감지하면 일정 시간 동안 청소 출력을 높여야 한다.
- 출력 증가 시간과 출력 수준은 추후 설계에서 조정 가능해야 한다.

## 4. Non-Functional Requirements

### 4.1 Modifiability

- 센서가 추가되거나 변경될 수 있으므로 센서 입력 처리는 확장 가능해야 한다.
- 청소 동작 정책은 향후 변경 가능성을 고려해 특정 하드웨어 구현에 강하게 결합되지 않아야 한다.

### 4.2 Maintainability

- 자동 청소 로직은 하드웨어 제어 세부 구현과 분리되어야 한다.
- 장애물 회피, 먼지 감지 대응, 이동 제어 판단은 이해하고 변경하기 쉬운 구조로 설계되어야 한다.

### 4.3 Extensibility

- 특정 지점을 일정 시간 선회하며 청소하는 기능을 추가할 수 있어야 한다.
- 모바일 앱과 통신하는 기능을 추가할 수 있어야 한다.
- 더 효율적인 청소를 위한 머신러닝 및 추론 기능을 추가할 수 있어야 한다.

## 5. Out of Scope

- 센서 하드웨어의 상세 설계 및 구현
- 모터, 브러시, 물걸레 장치 등 하드웨어 제어의 상세 구현
- 모바일 앱 자체의 구현
- 머신러닝 모델의 상세 설계 및 학습 구현

## 6. Future or Extended Requirements

- RVC는 센서를 추가하거나 변경할 수 있다.
- RVC는 한 지점을 일정 시간 선회하며 청소할 수 있다.
- RVC는 모바일 앱과 통신할 수 있다.
- RVC는 더 효율적인 청소를 위해 머신러닝과 추론을 수행할 수 있다.
